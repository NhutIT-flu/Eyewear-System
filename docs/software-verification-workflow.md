# Quy Trinh Kiem Chung Phan Mem - Eyewear System  ## 1. Muc tieu  Tai lieu nay mo ta quy trinh kiem chung phan mem cho du an Eyewear System. Quy trinh ket hop Jira, Git, Jenkins va mot so cong cu kiem thu/kiem tra chat luong nhu Postman, SonarQube va CodeceptJS.  Muc tieu cua quy trinh:  - Dam bao moi yeu cau tren Jira duoc gan voi code, test case va bang chung kiem thu. - Dam bao code duoc quan ly bang Git va co lich su thay doi ro rang. - Dam bao backend API duoc kiem thu bang Postman. - Dam bao code duoc kiem tra tu dong bang Jenkins. - Dam bao chat luong code co the duoc danh gia bang SonarQube. - Dam bao giao dien/chuc nang nguoi dung co the duoc kiem thu bang CodeceptJS neu can.  ## 2. Tong quan workflow  Quy trinh tong quat:  ```text Jira Ticket     -> Tao Git Branch     -> Code / Fix / Implement     -> Commit & Push     -> Jenkins CI     -> Static Check / Syntax Check     -> Postman API Test     -> SonarQube Code Quality Check     -> CodeceptJS UI/E2E Test     -> Attach Evidence vao Jira     -> Review     -> Done ```  Noi don gian:  - Jira quan ly viec can lam. - Git quan ly source code. - Jenkins tu dong chay quy trinh kiem tra. - Postman kiem thu REST API. - SonarQube kiem tra chat luong code. - CodeceptJS kiem thu thao tac nguoi dung tren giao dien.  ## 3. Vai tro cua tung cong cu  | Cong cu | Vai tro trong quy trinh | |---|---| | Jira | Quan ly Epic, Story, Task, Test Case, bug va trang thai cong viec | | Git/GitHub | Quan ly source code, branch, commit, pull request | | Jenkins | Tu dong chay pipeline CI sau khi push code | | Postman | Kiem thu REST API backend | | SonarQube | Phan tich chat luong code, bug, vulnerability, code smell | | CodeceptJS | Kiem thu UI/E2E tren trinh duyet | | MySQL | Luu tru du lieu test va du lieu ung dung |  ## 4. Quy trinh chi tiet  ## Buoc 1: Tao va phan tich Jira Ticket  Moi yeu cau can duoc tao tren Jira theo cau truc:  ```text Epic -> Story -> Task/Test Case ```  Vi du:  ```text Epic: Auth Module Story: Complete Login API Test Case: AUTH - Test Login Success Test Case: AUTH - Test Wrong Password Test Case: AUTH - Test Missing Email ```  Moi Jira Story can co:  - Mo ta yeu cau. - Acceptance Criteria. - API endpoint lien quan. - Nguoi phu trach. - Trang thai: To Do, In Progress, In Review, Done. - Evidence sau khi test.  Vi du acceptance criteria:  ```text - User can login with valid email and password. - API returns user information and token. - API rejects wrong password with 401. - API rejects missing email or password with 422/400. ```  ## Buoc 2: Tao Git Branch theo Jira Ticket  Khi bat dau lam mot ticket, developer tao branch tu `main` hoac `develop`.  Quy tac dat ten branch:  ```text feature/ES-11-login-api fix/ES-15-reset-password test/ES-46-auth-login-success ```  Vi du:  ```bash git checkout -b feature/ES-11-login-api ```  Muc dich:  - Tach rieng code cua tung ticket. - De truy vet ticket nao tao ra thay doi nao. - De reviewer de kiem tra.  ## Buoc 3: Code hoac sua chuc nang  Developer sua code theo dung module cua backend.  Vi du mapping trong du an Eyewear:  | Module | Controller | Service | |---|---|---| | Auth | `AuthController.php` | `AuthService.php` | | Product | `ProductController.php`, `CategoryController.php` | `CatalogService.php` | | Cart | `CartController.php` | `CartService.php` | | Checkout | `CheckoutController.php` | `CheckoutService.php` | | Payment | `PaymentController.php` | `PaymentService.php` | | Operations | `OperationsController.php` | `OperationsService.php` | | Dashboard | `DashboardController.php` | `DashboardService.php` | | Admin | `AdminController.php` | `AdminService.php` |  Nguyen tac:  - Controller nhan request va tra response. - Service xu ly logic nghiep vu. - Model/database xu ly du lieu. - Middleware kiem tra auth va permission.  ## Buoc 4: Commit code len Git  Sau khi sua code, developer commit theo convention.  Mau commit:  ```text type(scope): description ```  Vi du:  ```bash git add backend/app/Http/Controllers/Api/V1/AuthController.php git commit -m "feat(auth): complete login API" git push origin feature/ES-11-login-api ```  Mot so type thuong dung:  | Type | Y nghia | |---|---| | feat | Them chuc nang moi | | fix | Sua loi | | test | Them/sua test | | docs | Sua tai lieu | | refactor | Cai tien code khong doi hanh vi | | chore | Cau hinh, pipeline, viec phu tro |  ## Buoc 5: Jenkins CI tu dong kiem tra  Sau khi push code, Jenkins chay pipeline trong file:  ```text Jenkinsfile ```  Pipeline hien tai cua du an co cac stage:  1. Checkout source code. 2. Kiem tra cau truc project. 3. Kiem tra cu phap PHP neu co PHP CLI. 4. Kiem tra file frontend quan trong. 5. Kiem tra test folder va Postman collection. 6. Archive README, docs va Postman collection.  Ket qua:  - Neu pass: build mau xanh, co the tiep tuc test API. - Neu fail: developer doc console log, sua loi, push lai.  ## Buoc 6: Kiem thu REST API bang Postman  Postman duoc dung de kiem thu API backend.  Trong du an da co file:  ```text Eyewear-System.postman_collection.json ```  Can nam 3 khai niem trong Postman:  | Khai niem | Y nghia | |---|---| | Workspace | Noi lam viec chua collection, environment, team sharing | | Collection | Tap hop cac request API cua du an | | Environment | Tap bien cau hinh nhu `base_url`, `token`, `user_id`, `order_id` |  ## 6.1 Cau hinh Environment  Tao environment ten:  ```text Eyewear Local ```  Bien can co:  | Variable | Gia tri vi du | |---|---| | `base_url` | `http://localhost:8000/api/v1` | | `token` | token nhan sau khi login | | `product_id` | ID san pham dung de test | | `variant_id` | ID bien the san pham | | `cart_item_id` | ID item trong gio hang | | `order_id` | ID don hang | | `payment_id` | ID payment |  ## 6.2 Quy trinh test API bang Postman  Thu tu test nen di theo luong that cua he thong:  ```text Health Check     -> Register     -> Verify/Login     -> Product Listing     -> Product Detail     -> Add To Cart     -> Update Cart     -> Apply Voucher     -> Checkout     -> Payment Process     -> Order History     -> Dashboard/Ops/Admin ```  Moi request can ghi lai:  - Method. - Endpoint. - Request body. - Expected status code. - Actual status code. - Response body. - Screenshot hoac exported result.  Vi du:  ```text Test Case: AUTH - Test Login Success Method: POST Endpoint: {{base_url}}/auth/login Expected: 200 OK Evidence: Screenshot response co user va token ```  ## Buoc 7: Kiem tra chat luong code bang SonarQube  SonarQube dung de kiem tra:  - Bug. - Vulnerability. - Security hotspot. - Code smell. - Duplicate code. - Maintainability.  Website tham khao:  ```text https://github.com/SonarSource/sonarqube ```  ## 7.1 Cach ap dung vao du an  Vi du them file cau hinh:  ```text sonar-project.properties ```  Noi dung goi y:  ```properties sonar.projectKey=eyewear-system sonar.projectName=Eyewear System sonar.projectVersion=1.0 sonar.sources=backend,frontend sonar.exclusions=backend/PHPMailer/**,frontend/assets/images/** sonar.sourceEncoding=UTF-8 ```  Sau do Jenkins co the them stage:  ```text SonarQube Analysis ```  Ket qua SonarQube duoc dung lam evidence trong Jira:  ```text SonarQube scan completed. No blocker issue found. ```  ## Buoc 8: Kiem thu UI/E2E bang CodeceptJS  CodeceptJS dung de kiem tra giao dien va hanh vi nguoi dung tren browser.  Website tham khao:  ```text https://codecept.io/ ```  CodeceptJS phu hop de test cac luong:  - Login tren giao dien. - Tim san pham. - Them san pham vao gio hang. - Checkout. - Admin xem dashboard.  Vi du test case UI:  ```text Scenario: Customer can login successfully Given user opens login page When user enters valid email and password Then user is redirected to account/profile page ```  Luu y voi du an hien tai:  - Frontend dang la HTML/CSS/JavaScript tinh. - Chua co `package.json`, nen CodeceptJS chua duoc cai san. - Neu muon dung CodeceptJS, can tao cau hinh Node.js rieng cho test E2E.  ## Buoc 9: Gan evidence vao Jira  Sau khi kiem thu xong, tester cap nhat Jira ticket.  Moi ticket nen co evidence:  - Link commit Git. - Link Jenkins build. - Screenshot Postman response. - Screenshot SonarQube result neu co. - Screenshot UI test result neu co. - Ghi expected result va actual result.  Mau ghi comment Jira:  ```text Verification completed.  Git branch: feature/ES-11-login-api Commit: <commit_hash> Jenkins build: #15 - Passed Postman test: Login Success - Passed Expected status: 200 Actual status: 200 Evidence: attached screenshot ```  ## Buoc 10: Review va chuyen trang thai Jira  Trang thai Jira nen di theo quy trinh:  ```text To Do -> In Progress -> In Review -> Done ```  Y nghia:  | Trang thai | Y nghia | |---|---| | To Do | Chua lam | | In Progress | Dang code hoac dang test | | In Review | Da lam xong, cho review/test lai | | Done | Da pass test va co evidence |  Ticket chi duoc chuyen Done khi:  - Code da push len Git. - Jenkins build pass. - Postman/API test pass. - Bug nghiem trong da duoc sua. - Evidence da duoc attach vao Jira.  ## 5. Quy trinh kiem chung theo module backend  ## 5.1 Auth Module  Workflow:  ```text Register -> Verify Email -> Login -> Get Profile -> Change Password -> Logout ```  Can test:  - Register success. - Duplicate email. - Login success. - Wrong password. - Missing email/password. - Unauthorized access. - Permission denied.  ## 5.2 Product Module  Workflow:  ```text Category -> Product Listing -> Product Detail -> Search -> Filter -> Admin CRUD -> Inventory Update ```  Can test:  - Product listing. - Product detail. - Product search. - Product filter. - Category API. - Create/update/deactivate product. - Inventory update.  ## 5.3 Cart and Checkout Module  Workflow:  ```text Login -> Add To Cart -> Update Quantity -> Apply Voucher -> Review Total -> Checkout ```  Can test:  - Add item. - Add out-of-stock item. - Remove item. - Update quantity. - Apply valid/invalid voucher. - Checkout empty cart. - Checkout success.  ## 5.4 Payment and Order Module  Workflow:  ```text Checkout -> Create Order -> Create Payment -> Check Payment Status -> Confirm Payment -> View Order History ```  Can test:  - Order creation. - Payment process. - Payment status. - Payment confirmation. - Order history. - Order detail.  ## 5.5 Operations and Dashboard Module  Workflow:  ```text Staff Login -> View Ops Orders -> Advance Production -> Create Shipment -> Update Tracking -> Dashboard Report ```  Can test:  - Operations order list. - Advance production step. - Create shipment. - Update shipment. - Dashboard summary. - Revenue analytics. - Product statistics.  ## 6. Definition of Done cho kiem chung  Mot story/test case duoc xem la xong khi:  - Yeu cau tren Jira ro rang. - Code da commit va push len Git. - Jenkins build pass. - API test Postman pass. - Neu co, SonarQube khong co blocker/critical issue. - Neu co UI flow, CodeceptJS test pass. - Evidence da duoc attach vao Jira. - Reviewer chap nhan.  ## 7. Mau bang kiem chung de dua vao bao cao  | ID | Module | Cong cu | Noi dung kiem chung | Evidence | |---|---|---|---|---| | V-01 | Project | Jenkins | Kiem tra cau truc project | Jenkins build log | | V-02 | Backend | Jenkins/PHP | Kiem tra syntax PHP | Console output | | V-03 | Auth | Postman | Login/Register/Auth token | Postman screenshot | | V-04 | Product | Postman | Listing/Detail/Search/Filter | Postman screenshot | | V-05 | Cart | Postman | Add/Update/Remove/Checkout | Postman screenshot | | V-06 | Payment | Postman | Order/Payment/Confirm | Postman screenshot | | V-07 | Code Quality | SonarQube | Bug, vulnerability, code smell | SonarQube report | | V-08 | UI/E2E | CodeceptJS | Login, cart, checkout UI flow | Test report |  ## 8. Ket luan  Quy trinh kiem chung cua Eyewear System khong chi la chay test rieng le. Day la mot chuoi lien ket giua Jira, Git, Jenkins va cac cong cu kiem thu.  Quy trinh dung nen la:  ```text Jira ghi yeu cau Git quan ly code Jenkins tu dong kiem tra Postman kiem thu API SonarQube kiem tra chat luong code CodeceptJS kiem thu giao dien neu can Evidence gan lai Jira Reviewer xac nhan Ticket moi duoc Done ```
